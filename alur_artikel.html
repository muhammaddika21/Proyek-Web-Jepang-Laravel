Alur Sistem Artikel: Nihon Learn

Dokumentasi lengkap aliran data artikel dari Backend (Admin) ke Client Side (User)

Diagram di atas menunjukkan bagaimana data bergerak dari input admin hingga ditampilkan ke user.

2. Struktur Data (Database)

Artikel disimpan dalam tabel articles dengan field utama sebagai berikut:

Kategori Kolom Deskripsi
Metadata title, slug Judul artikel dan URL ramah SEO.
type Jenis artikel: umum atau bahasa.
status draft (belum terbit) atau published (bisa dilihat user).
Konten Inti content / grammar_explanation Isi artikel dari Quill Editor (HTML string).
vocabulary_list, quiz_questions Disimpan dalam format JSON (Array).
Media cover_image, additional_images Path gambar yang disimpan di folder storage public.
audio_file, youtube_url File audio pelafalan dan link video referensi.

3. Alur Backend (Admin Side)

1

Input Form Artikel

Admin mengisi form (Umum/Bahasa). Data seperti teks diproses oleh Quill Editor. Kosakata dan Kuis dimasukkan via form dinamis berbasis JavaScript. File media (cover, audio, gambar tambahan) dipilih dari perangkat.

2

Proses Controller (ArticleController@store/update)

  • Memvalidasi input agar sesuai kriteria.
  • Meng-upload file media ke storage/app/public/articles/... dan mengambil string path-nya.
  • Mendeteksi jika artikel adalah "Bahasa", konten dipindah ke kolom grammar_explanation.
  • Memparsing array input form ke bentuk JSON untuk vocabulary_list dan quiz_questions.
  • Menyimpan record ke database.

4. Alur Client Side (User View)

1

Controller Mengambil Data

Saat user membuka halaman belajar (misal via BahasaController@show), sistem mencari artikel berdasarkan slug yang berstatus published.

2

Rendering dengan Shortcode (Blade)

Pada tampilan (View), sistem membaca isi konten utama artikel. Karena sebelumnya Anda mengimplementasikan shortcode, tag seperti [vocab1] atau [gambar1] diganti menjadi komponen HTML interaktif. Ini menggabungkan data text dengan data JSON (seperti vocabulary_list) dan relasi media Storage menjadi satu halaman utuh.

3

Interaksi User

User dapat membaca konten, mendengarkan audio melalui web player, melihat gambar referensi, dan mengerjalan latihan dari data kuis yang dirender menggunakan JavaScript/Alpine di frontend.

Dibuat secara otomatis untuk proyek Nihon Learn.
