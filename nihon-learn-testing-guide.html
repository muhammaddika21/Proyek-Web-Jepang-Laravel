Modul 1 — Persiapan & Setup
Pastikan lingkungan dev berjalan sebelum mulai
4 test
Server Laravel berjalan
Jalankan php artisan serve lalu akses ⬡ localhost:8000 — harus muncul halaman Home, bukan error.
Critical
Database terhubung
Pastikan .env sudah diatur ke DB_DATABASE=nihon_learn. Akses ⬡ localhost:8000 tidak boleh muncul error "SQLSTATE[HY000] Unknown database".
Critical
Vite / Asset build berjalan
Jalankan npm run dev secara terpisah. Pastikan CSS TailwindCSS ter-load dengan benar (tidak ada unstyled HTML) di semua halaman.
High
Storage symlink aktif
Jalankan php artisan storage:link. Verifikasi gambar/media yang sudah diupload bisa tampil di frontend (tidak broken image icon).
High

Modul 2 — Navigasi & UI Global
Navbar, scroll behavior, responsive layout
7 test
Navbar transparan → putih saat scroll
Buka halaman Home. Saat di atas, navbar harus transparan. Saat scroll lebih dari 50px, navbar harus berubah menjadi bg-white/95 dengan shadow.
High
Tidak ada layout shift saat scroll
Scroll perlahan ke bawah di halaman mana saja. Teks navigasi (Home, Bahasa, dll) dan tombol Login tidak boleh bergeser ke kiri/kanan saat transisi warna navbar terjadi.
✅ Cara verifikasi
  1. Buka DevTools (F12) → tab Performance → Record lalu scroll
  2. Cari Layout Shift di timeline. Nilai CLS harus < 0.1
High
Semua link navigasi berfungsi
Klik satu per satu: Home → Bahasa → Kegiatan → Budaya → Logo. Masing-masing harus mengarah ke halaman yang tepat tanpa error 404.
Critical
Hamburger menu mobile berfungsi
Di viewport < 768px (gunakan DevTools Responsive Mode), klik ikon hamburger. Menu mobile harus muncul dengan animasi. Klik salah satu link, menu harus tertutup otomatis.
Medium
Font Google Fonts ter-load
Periksa apakah font Zen Kurenaido (logo), Noto Sans JP (body), dan Sawarabi Mincho (tombol kategori) tampil dengan benar, bukan fallback sans-serif generik.
Medium
Tombol Login mengarah ke halaman login admin
Klik tombol Login di pojok kanan navbar. Harus diarahkan ke ⬡ /admin (atau /login jika belum login).
High
Tidak ada horizontal scroll (overflow-x)
Di semua halaman, tidak boleh ada scrollbar horizontal yang tidak diinginkan. Verifikasi di layar lebar (1920px) dan mobile (375px).
Medium

Modul 3 — Halaman Publik (Frontend)
Home, Bahasa, Budaya, Kegiatan, dan halaman Detail
10 test
Halaman Home load tanpa error
Akses ⬡ localhost:8000. Pastikan hero section, konten artikel preview, dan footer tampil sempurna. Console di DevTools tidak boleh ada error merah.
Critical
Halaman Bahasa — Filter kategori berfungsi
Buka ⬡ /bahasa. Klik tombol kategori (Kanji, Kotoba, dll). Konten artikel harus beralih secara instan tanpa reload halaman.
Langkah verifikasi
  1. Klik "Kanji" → hanya artikel Kanji yang tampil
  2. Klik "Semua" → semua artikel muncul kembali
  3. Filter level (Pemula/Menengah/Mahir) juga harus berfungsi
Critical
Halaman Bahasa — Search artikel real-time
Ketik kata kunci di kotak search. Artikel harus difilter secara langsung (dalam 200ms). Ketik kata yang tidak ada → muncul pesan "Tidak ada hasil".
High
Halaman Budaya — Tombol kategori tepat di tengah
Buka ⬡ /budaya. Filter bar harus menampilkan search di kiri (max 280px), tombol kategori tepat di tengah layar secara visual, menggunakan palet warna #404235.
High
Halaman Kegiatan — Filter & warna konsisten
Buka ⬡ /kegiatan. Sub-hero harus berwarna caramel #d6975e. Filter bar identik strukturnya dengan halaman Budaya.
High
Placeholder kartu artikel muncul jika tidak ada gambar
Artikel tanpa cover_image harus menampilkan placeholder gradien berwarna (bukan ikon gambar rusak/broken).
Medium
Navigasi paginasi berfungsi
Jika ada lebih dari 9 artikel dalam satu kategori, tombol paginasi harus muncul. Klik halaman berikutnya, konten berganti dan halaman tidak reload penuh.
Medium
Halaman detail artikel Bahasa terbuka
Klik salah satu artikel di halaman Bahasa. Harus terbuka halaman detail dengan konten lengkap — judul, cover image / placeholder, body text, tabel kosakata (jika ada), dan kuis (jika ada).
Elemen yang harus dicek
  1. Render konten HTML dari Quill editor
  2. Tabel kosakata (vocab) tampil dengan styling yang benar
  3. Badge level (Pemula/Menengah/Mahir) tampil
  4. Jika ada kuis → tombol "Cek Jawaban" hanya aktif setelah semua soal dijawab
Critical
Halaman detail artikel Budaya/Kegiatan terbuka
Klik artikel di halaman Budaya atau Kegiatan. Konten harus tampil, termasuk gambar tambahan (additional media) jika ada, dan embed Youtube jika URL diisi.
High
Responsivitas di mobile (375px)
Gunakan DevTools (F12 → Toggle device toolbar → iPhone SE). Semua halaman publik harus bisa digunakan dengan nyaman — tidak ada teks overflow, tidak ada elemen yang terpotong.
High

Modul 4 — Autentikasi Admin
Login, proteksi route, dan logout
5 test
⚠️ PENTING: Sebelum mulai, pastikan ada user admin di tabel users. Jalankan php artisan db:seed jika belum ada.
Halaman login tampil dengan benar
Buka ⬡ /login. Form login harus tampil dengan field email dan password. Styling Nihon-Learn harus teraplikasi.
High
Login berhasil dengan kredensial benar
Masukkan email dan password yang valid. Ekspektasi: Redirect ke /admin (Dashboard). Session harus aktif.
Critical
Login gagal dengan kredensial salah
Masukkan password yang salah. Ekspektasi: Tetap di halaman /login dan muncul pesan error validasi — tidak boleh crash atau error 500.
Critical
Proteksi route admin terhadap guest
Tanpa login, akses langsung ke ⬡ /admin/articles. Ekspektasi: Di-redirect ke halaman login, bukan muncul halaman admin.
Critical
Logout berfungsi
Setelah login, klik tombol Logout. Ekspektasi: Session dihapus, diredirect ke halaman login. Jika langsung akses /admin setelahnya, harus balik ke login.
High

Modul 5 — Admin Dashboard & Manajemen Artikel
Daftar artikel, filter dinamis, statistik
6 test
Dashboard menampilkan statistik yang benar
Login lalu buka ⬡ /admin. Card statistik (total artikel, artikel terpublished, dll) harus menampilkan angka yang konsisten dengan data di database.
High
Daftar artikel tampil di /admin/articles
Semua artikel (published dan draft) harus tampil dalam tabel dengan info: Judul, Tipe, Kategori, Status, Tanggal.
High
Filter tipe artikel → kategori dinamis berubah
Di halaman daftar artikel admin, ubah dropdown Tipe Artikel:
Skenario yang harus lolos
  1. Pilih "Bahasa" → dropdown Kategori hanya menampilkan kategori bahasa (Kanji, Kotoba, Bunpou, Kaiwa, dll)
  2. Pilih "Umum" → dropdown Kategori hanya menampilkan kategori umum (Budaya, Kegiatan, dll)
  3. Ganti tipe → pilihan kategori yang aktif sebelumnya direset ke "Semua Kategori"
Critical
Search artikel di admin berfungsi
Ketik judul artikel di kotak search, lalu klik tombol Filter. Daftar artikel harus mengecil sesuai kata kunci.
Medium
Toggle publish/draft berfungsi
Di daftar artikel, klik tombol toggle Status pada suatu artikel. Status harus berganti antara Published → Draft tanpa reload halaman penuh. Verifikasi perubahan status di database.
High
Preview artikel dari admin berfungsi
Klik tombol Preview / Eye icon pada artikel. Harus membuka halaman detail artikel (tampilan publik) yang sesuai.
Medium

Modul 6 — CRUD Artikel (Create, Read, Update, Delete)
Alur pembuatan, pengeditan, dan penghapusan artikel
7 test
Buat artikel Umum (Budaya/Kegiatan) baru
Buka ⬡ /admin/articles/create dan isi form:
Field yang wajib diisi & diverifikasi
  1. Judul artikel (wajib)
  2. Kategori (pilih Budaya/Kegiatan)
  3. Konten melalui Quill editor (wajib)
  4. Cover image (opsional)
  5. Status: Published
  6. Klik Simpan → harus redirect ke daftar artikel dengan pesan sukses
Critical
Buat artikel Bahasa baru (form khusus)
Buka ⬡ /admin/articles/create-bahasa dan isi:
Field wajib untuk artikel Bahasa
  1. Judul, Kategori, JLPT Level (wajib)
  2. Konten teks (wajib)
  3. Tabel kosakata (vocab table) — tambah minimal 1 baris
  4. Soal kuis — tambah minimal 1 soal
  5. Simpan → verifikasi di frontend /bahasa artikel muncul
Critical
Validasi form — field wajib tidak boleh kosong
Coba simpan artikel tanpa mengisi judul atau konten. Ekspektasi: Form menampilkan pesan validasi merah pada field yang kosong — tidak boleh artikel tersimpan dengan data kosong.
Critical
Upload cover image berhasil
Saat membuat artikel, upload gambar (JPG/PNG, < 2MB). Setelah disimpan, gambar harus tampil sebagai cover di halaman daftar dan detail artikel.
High
Edit artikel yang sudah ada
Klik tombol Edit pada artikel yang sudah ada. Form harus terisi otomatis dengan data lama. Ubah judul → simpan → verifikasi perubahan tampil di frontend.
Critical
Artikel published muncul di frontend
Artikel yang di-set Published harus tampil di halaman publik (/bahasa, /budaya, /kegiatan). Artikel berstatus Draft tidak boleh tampil di frontend.
Critical
Hapus artikel berfungsi
Klik tombol Delete pada artikel. Muncul konfirmasi dialog. Setujui → artikel terhapus dari daftar dan hilang dari frontend. Verifikasi artikel tidak bisa diakses via URL slug-nya.
High

Modul 7 — Skenario Negatif & Edge Case
Uji batas-batas sistem dan penanganan error
3 test
ℹ️ Modul ini menguji bagaimana sistem merespons input yang tidak valid atau akses yang tidak sah.
Akses artikel dengan slug tidak valid → 404
Coba akses ⬡ /bahasa/slug-tidak-ada. Ekspektasi: Tampil halaman 404 yang sesuai desain, bukan error Laravel mentah.
High
Kuis: tombol "Cek Jawaban" hanya aktif setelah semua soal dijawab
Buka halaman detail artikel Bahasa yang memiliki kuis. Coba klik "Cek Jawaban" sebelum semua soal dijawab. Ekspektasi: Tombol non-aktif atau muncul peringatan "Harap jawab semua soal terlebih dahulu".
High
Upload file berformat tidak valid ditolak
Coba upload file .pdf atau .exe sebagai cover image. Ekspektasi: Validasi server menolak dan menampilkan pesan error — file tidak tersimpan di storage.
Medium

Catatan Akhir & Checklist Pre-Launch
Hal-hal yang perlu diperhatikan sebelum deploy ke production
🚨 Hapus route /bypass di routes/web.php sebelum deploy ke production! Route ini memungkinkan siapapun masuk sebagai admin tanpa password.

⚠️ Set APP_DEBUG=false dan APP_ENV=production di file .env production. Jangan pernah menampilkan stack trace error ke publik.

💡 Jalankan php artisan optimize dan php artisan view:cache sebelum deploy untuk performa yang optimal.

Perintah Fungsi
php artisan migrate:fresh --seed Reset & isi ulang database dari awal (hati-hati di production!)
php artisan storage:link Buat symlink public/storage ke storage/app/public
php artisan cache:clear Bersihkan semua cache aplikasi
php artisan config:clear Bersihkan cache konfigurasi
php artisan route:list Tampilkan semua route yang terdaftar
npm run build Build asset untuk production (bukan dev)
