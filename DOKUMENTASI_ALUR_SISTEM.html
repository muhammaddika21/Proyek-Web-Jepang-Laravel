1 Gambaran Besar Sistem (Arsitektur)

NihonLearn adalah aplikasi web dengan dua area utama: area publik untuk pengunjung biasa, dan area admin untuk pengelola konten. Keduanya berjalan dalam satu proyek Laravel tetapi menggunakan tampilan (CSS/Layout) yang sepenuhnya terpisah.

🌍 Area Publik (Pengunjung)

  • URL: /, /bahasa
  • Layout: layouts/app.blade.php
  • CSS: app.css (Tailwind standar)
  • Tidak perlu login
  • Menampilkan konten UKM

🛡️ Area Admin (Pengelola)

  • URL: /admin/*
  • Layout: layouts/admin.blade.php
  • CSS: admin.css (TailAdmin theme)
  • Harus login terlebih dahulu
  • Mengelola artikel & konten
💡 Kenapa dipisah? CSS TailAdmin memiliki ratusan custom theme (warna, font, breakpoint) yang akan merusak tampilan landing page UKM jika digabung. Dengan memisahkan menjadi app.css dan admin.css, keduanya bisa hidup berdampingan tanpa konflik.

3 Sistem Dual CSS: app.css vs admin.css

Ini adalah perubahan paling krusial yang kita lakukan. File vite.config.js dikonfigurasi untuk mengompilasi dua file CSS terpisah:

// vite.config.js
laravel({
    input: [
        'resources/css/app.css',    // ← CSS untuk halaman publik
        'resources/css/admin.css',  // ← CSS untuk halaman admin
        'resources/js/app.js',
    ],
})

📄 app.css (Publik)

  • Hanya berisi: @import 'tailwindcss'
  • Tailwind v4 standar, bersih
  • Tidak ada custom theme TailAdmin
  • Digunakan di layouts/app.blade.php

📄 admin.css (Admin)

  • 877 baris CSS TailAdmin
  • Custom font Outfit, custom breakpoint
  • Tema warna brand, dark mode
  • Digunakan di layouts/admin.blade.php
⚠️ Jangan gabungkan! Jika admin.css dimasukkan ke app.css (seperti yang terjadi sebelumnya), seluruh tampilan landing page UKM akan rusak karena theme TailAdmin menimpa semua warna dan font standar Tailwind.

// AuthController.php - fungsi login()
public function login(Request $request) {

    // 1. Validasi: username & password wajib diisi
    $credentials = $request->validate([
        'username' => ['required'],
        'password' => ['required'],
    ]);

    // 2. Cek ke database: apakah cocok?
    if (Auth::attempt($credentials)) {
        $request->session()->regenerate(); // Buat sesi baru (keamanan)
        return redirect()->intended('/admin'); // ← REDIRECT KE ADMIN!
    }

    // 3. Jika gagal: kembali dengan pesan error
    return back()->withErrors([
        'username' => 'Username atau password salah.',
    ])->onlyInput('username');
}

6 Sistem Blade: Layout, Include, Section, Yield

Laravel Blade adalah "bahasa template" yang memungkinkan kita memecah HTML menjadi bagian-bagian kecil yang bisa di-reuse. Analoginya seperti puzzle.

Konsep @extends + @section

File konten "meminjam" kerangka dari file layout menggunakan @extends, lalu mengisi bagian kosong dengan @section:

File Kerangka (layouts/admin.blade.php)

<body>
  @include('layouts.admin-sidebar')
  @include('layouts.admin-header')
  <div class="p-6">
    <!-- KOTAK KOSONG -->
    @yield('content')
  </div>
</body>

File Konten (admin/dashboard-new.blade.php)

<!-- Pakai kerangka admin -->
@extends('layouts.admin')

<!-- Isi kotak yang kosong tadi -->
@section('content')
  <h1>Selamat Datang!</h1>
  <div>... grafik, tabel ...</div>
@endsection

Konsep @include

Untuk "menarik" file lain dan menyatukannya. Seperti copy-paste tapi lebih rapi:

@include('layouts.admin-sidebar')  ← Tarik konten sidebar ke sini
@include('layouts.admin-header')   ← Tarik konten header ke sini
@include('layouts.admin-sidebar-widget')  ← Widget di bawah sidebar

@stack + @push (untuk JavaScript)

Mirip seperti @yield tapi khusus untuk script yang perlu ditambahkan dari file konten:

<!-- Di layouts/admin.blade.php -->
@stack('scripts')  ← Tampung script dari konten

<!-- Di dashboard-new.blade.php -->
@push('scripts')
  <script>console.log('Script khusus halaman ini')</script>
@endpush
7 Alpine.js: Interaktivitas Tanpa jQuery

Template admin menggunakan Alpine.js — library JavaScript ringan untuk menangani interaksi UI seperti dropdown, toggle, dan show/hide elemen. Cara penulisannya langsung di HTML.

💡 Kenapa Alpine.js? Alpine.js jauh lebih ringan dari Vue.js atau React, dan tidak perlu dikompilasi. Cocok untuk interaksi kecil-kecilan di halaman admin seperti dropdown, accordion, dan sidebar toggle.

Kata Kunci Penting Alpine.js:

SintaksFungsiAnalogi
x-dataMendefinisikan data/state lokalSeperti menyiapkan variabel
x-showTampilkan/sembunyikan elemenSeperti display:block/none otomatis
@clickJalankan kode ketika diklikSeperti onclick= di HTML biasa
x-bind / :Ikat nilai atribut ke variabelNilai berubah sesuai kondisi
x-transitionAnimasi masuk/keluar elemenFade in/out otomatis
$storeState yang diakses semua komponenVariabel global

Contoh Nyata: Dropdown Profil User

<!-- user-dropdown.blade.php -->
<div x-data="{ dropdownOpen: false }">

    <!-- Klik tombol → ubah nilai dropdownOpen -->
    <button @click="dropdownOpen = !dropdownOpen">
        Profil Saya
    </button>

    <!-- Hanya tampil jika dropdownOpen bernilai true -->
    <div x-show="dropdownOpen" x-transition>
        <a href="/admin">Edit Profile</a>
        <a href="/logout">Sign Out</a>
    </div>
</div>

Contoh Nyata: Sidebar Collapse (Alpine Store)

Sidebar admin menggunakan Alpine.store — sebuah state global yang bisa diakses dari mana saja (tidak hanya dari komponen yang sama).

<!-- Definisi di layouts/admin.blade.php -->
Alpine.store('sidebar', {
    isExpanded: window.innerWidth >= 1280, // Default terbuka di layar lebar
    
    toggleExpanded() {
        this.isExpanded = !this.isExpanded; // Toggle buka/tutup
    }
});

<!-- Penggunaan di admin-sidebar.blade.php -->
<aside :class="{
    'w-[290px]': $store.sidebar.isExpanded,  // Lebar jika terbuka
    'w-[90px]': !$store.sidebar.isExpanded   // Sempit jika tertutup
}">
8 MenuHelper: Mengatur Navigasi Sidebar

File app/Helpers/MenuHelper.php adalah sumber kebenaran untuk menu di sidebar admin. Daripada menulis HTML menu langsung di sidebar (yang sulit diubah), kita mendefinisikan menu sebagai array PHP lalu me-loop-nya di Blade.

// app/Helpers/MenuHelper.php
public static function getMainNavItems(): array {
    return [
        [
            'icon'  => 'dashboard',  // Nama ikon SVG
            'name'  => 'Dashboard',
            'path'  => '/admin',
        ],
        [
            'icon'     => 'forms',
            'name'     => 'Buat Artikel',
            'subItems' => [  // Menu dengan submenu
                ['name' => 'Artikel Umum',  'path' => '/admin/articles/create'],
                ['name' => 'Artikel Bahasa', 'path' => '/admin/articles/create-bahasa'],
            ],
        ],
    ];
}
✅ Cara menambah menu baru: Cukup tambahkan satu array baru di dalam fungsi getMainNavItems() di file MenuHelper.php. Menu langsung muncul di sidebar tanpa perlu sentuh file HTML sidebar sama sekali.
9 Middleware: Sistem Keamanan Halaman Admin

Middleware adalah "petugas keamanan" yang bekerja sebelum request sampai ke controller. Ada dua middleware utama yang dipakai:

🔒 Middleware 'auth'

  • Dipakai di semua rute /admin/*
  • Cek: apakah user sudah login?
  • Jika BELUM login → redirect ke /login
  • Jika SUDAH login → lanjutkan ke halaman

👤 Middleware 'guest'

  • Dipakai di rute /login
  • Cek: apakah user BELUM login?
  • Jika SUDAH login → redirect ke /admin
  • Jika BELUM login → tampilkan halaman login
// routes/web.php

// Halaman login hanya untuk yang BELUM login
Route::middleware('guest')->group(function () {
    Route::get('/login', [AuthController::class, 'showLoginForm']);
    Route::post('/login', [AuthController::class, 'login']);
});

// Semua halaman admin hanya untuk yang SUDAH login
Route::middleware('auth')->prefix('admin')->group(function () {
    Route::get('/', ...);         // /admin
    Route::get('/articles', ...); // /admin/articles
});
10 Ringkasan: Apa yang Harus Dipahami

Sebelum lanjut ke pembuatan Backend Database, pastikan Anda paham konsep-konsep berikut:

✅ Harus Paham

  • Alur Route → Controller → View
  • Perbedaan app.css vs admin.css
  • @extends, @section, @yield, @include
  • Cara kerja middleware 'auth' dan 'guest'
  • Cara menambah menu di MenuHelper.php
  • Logika login di AuthController.php

📚 Pelajari Lebih Lanjut

  • Alpine.js: x-data, x-show, @click
  • Alpine.js: $store (global state)
  • Blade Components (x-component)
  • Laravel Eloquent (untuk backend nanti)
  • Database Migration & Seeder
🚀 Langkah Berikutnya: Backend Database Setelah memahami alur di atas, langkah berikutnya adalah membuat tabel database untuk Artikel menggunakan Migration, mendefinisikan model Artikel, dan membuat Controller dengan fungsi CRUD (Create, Read, Update, Delete) yang terhubung ke halaman-halaman admin yang sudah ada.
